═══════════════════════════════════════════════════════════════════════════════
                            THREAD ARCHITECTURE
═══════════════════════════════════════════════════════════════════════════════

Main Thread:
│
├─► Voice Interface Thread (blocking)
│   └─► Listens for voice commands
│   └─► Processes speech recognition
│   └─► Speaks TTS responses
│
├─► Vision Monitor Thread (daemon)
│   └─► Captures frames @ 5 FPS
│   └─► Runs face detection
│   └─► Emits ACTIVITY_CLUE events
│
├─► Sensor Monitor Thread (daemon)
│   └─► Polls ESP32 every 5s
│   └─► Parses temperature/humidity
│   └─► Emits ENV_UPDATE events
│
├─► Time Tick Thread (daemon)
│   └─► Emits TIME_TICK every 1s
│   └─► Triggers freshness decay
│
└─► Reasoning Loop Thread (daemon)
    └─► Consumes events from queue
    └─► Updates world state
    └─► Evaluates decisions
    └─► Proposes actions
    └─► Executes callbacks


═══════════════════════════════════════════════════════════════════════════════
                             DATA FLOW EXAMPLE
═══════════════════════════════════════════════════════════════════════════════

User looks away from screen:

1. [Vision Thread]
   Camera frame → No face detected → confidence: 70%
   Emit: EV_ACTIVITY_CLUE(value=1, confidence=70)

2. [Event Queue]
   Event added to queue

3. [Reasoning Thread]
   Event consumed from queue

4. [Event Reducer]
   world.activity_mode = DISTRACTED
   world.distraction_confidence = 70

5. [Decision Gate]
   belief: 80, freshness: 90
   activity_mode: DISTRACTED
   → permitted: TRUE, reason: "High distraction detected"

6. [Action Intent]
   Propose: SUGGEST_BREAK
   Rationale: "Distraction detected (70% confidence)"
   Priority: 7

7. [LLM Advisor]
   Context: World state + Decision + Intent
   Phi-2 generates:
   "You've been distracted for several minutes. A 5-minute break might help."

8. [Action Execution]
   callback: on_break_suggestion()
   TTS speaks the suggestion

9. [Cooldown]
   Next action blocked for 15 seconds

═══════════════════════════════════════════════════════════════════════════════
                            EXTENSIBILITY POINTS
═══════════════════════════════════════════════════════════════════════════════

1. NEW INPUT SOURCES
   Add thread → Generate events → Emit to queue
   Example: Calendar events, keyboard activity, mouse tracking

2. NEW EVENT TYPES
   Add to EventType enum → Handle in EventReducer
   Example: EV_CALENDAR_ALERT, EV_KEYBOARD_IDLE

3. NEW ACTIONS
   Define action ID → Implement in ActionIntent → Register callback
   Example: "SEND_NOTIFICATION", "ADJUST_LIGHTING"

4. CUSTOM POLICIES
   Modify DecisionGate.evaluate()
   Example: Time-of-day rules, user preferences

5. ALTERNATIVE LLMs
   Replace Phi-2 with any HuggingFace model
   Example: Mistral-7B, Llama-2, GPT4All

6. PERSISTENCE
   Add belief_logger → Log to database
   Example: SQLite for world state history
